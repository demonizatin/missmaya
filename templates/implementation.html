What gets sent where

/chat system = build_system_prompt() (Maya's 28 rules + JSON spec). User-1 = build_first_message_user_prompt() = profile + cross-session memory + in-session summary + today. Then last 30 messages, then the new user message. Streams SSE deltas.
background After every AI reply, lazy session_summary LLM call — only fires when the conversation has more than 30 messages, and summarizes only the trimmed-out portion (60 words).
/end_session Fires only if a client-side MEMORY_SIGNALS regex matched in the user message or AI reply. Calls memory_merge prompt → JSON-patch of facts/events → apply_memory_patch() with identity protection, regex fallback, and pending-merges queue.
/tts Server-side Piper neural TTS (ONNX). Female voice, streamed as audio chunks. Plays only on aiState === "speaking"; the Maya video gates on the same state.
storage memory_store/<user>.json facts+events buckets · _session_summaries/ rolling summary · _prompts_overrides.json admin edits · _pending_merges/ retry queue · _logs/.
backends CLI (subscription, default) · Anthropic API (key) · AWS Bedrock (Claude or qwen.qwen3-32b-v1:0). Picked at request time — system prompt and message format are identical across backends.
events Events stored with absolute date YYYY-MM-DD. Auto-archived 14 days after the date passes. Maya translates back to natural speech ("tomorrow", "next week") using today's date in the prompt.
